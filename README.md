# MARS Deepfake Audio Detection

This repository contains a complete pipeline for Deepfake Audio Detection, developed for the MARS Open Projects 2026.

## 📌 Project Description
The goal of this project is to classify speech recordings as either **Genuine (Human)** or **Deepfake (AI-Generated)**.
Advances in generative AI have enabled highly realistic synthetic speech. Detecting whether an audio recording is genuine or artificially generated is an important research and security challenge.

This system achieves high accuracy and reliability using a Convolutional Neural Network (CNN) trained on audio spectrograms extracted from the **Fake-or-Real (FoR)** dataset.

## 🔬 Methodology & Pipeline

1. **Dataset Handling:** 
   - The dataset used is the `for-norm` version of the FoR dataset, which balances gender and class, and normalizes sample rate, volume, and channels.
   - The data is split into `training`, `validation`, and `testing` sets.

2. **Preprocessing & Feature Extraction:**
   - Audio files (`.wav`) are read and decoded using TensorFlow's native audio operations (`tf.audio`).
   - Waveforms are padded or truncated to a fixed length of 3 seconds (48,000 samples at 16kHz).
   - **Short-Time Fourier Transform (STFT)** is applied to extract the magnitude spectrogram. This 2D representation captures both frequency and time information, making it ideal for CNNs.

3. **Model Architecture:**
   - A 2D **Convolutional Neural Network (CNN)** is employed.
   - The network consists of a resizing layer (to normalize spectrogram dimensions), followed by three Convolutional blocks (Conv2D + ReLU + MaxPooling2D).
   - The flattened features are passed through dense layers with Dropout for regularization.
   - The output layer uses a Sigmoid activation function for binary classification (0: Deepfake, 1: Genuine).

4. **Training Pipeline:**
   - The model is trained using the Adam optimizer and Binary Crossentropy loss.
   - Early Stopping is used to prevent overfitting, monitoring validation loss.
   - The best model weights are saved automatically.

## 📊 Evaluation Metrics

The system was evaluated against the following strict criteria:
- **Overall Accuracy:** ≥ 80%
- **Equal Error Rate (EER):** ≤ 12%
- **F1 Score:** ≥ 80%
- **Per-Class Accuracy:** ≥ 75%

*Note: You can view the exact metrics generated by running `python evaluate.py`, which will output a `report.md` file containing the Confusion Matrix and final scores.*

## 🚀 How to Run

### 1. Prerequisites
Install the required dependencies:
```bash
pip install tensorflow librosa streamlit scikit-learn matplotlib seaborn
```

### 2. Training the Model
To train the model from scratch on your local dataset:
```bash
python train.py
```
This will read data from `C:\mars ai\datasets\for-norm\training` and save the trained model to `model/best_model.keras`.

### 3. Evaluating the Model
To evaluate the model on the testing set and generate the performance report:
```bash
python evaluate.py --test_dir "C:\mars ai\datasets\for-norm\testing"
```

### 4. Predicting a Single File
You can test the model on any `.wav` file using the prediction script:
```bash
python predict.py "path/to/audio/file.wav"
```

### 5. Running the Web App (Streamlit)
To launch the interactive Deepfake Audio Detector web application:
```bash
streamlit run app.py
```
Upload an audio file in the browser interface to see the prediction and confidence score!

## 📁 Repository Structure
- `train.py`: Data pipeline and model training script.
- `evaluate.py`: Evaluation and metrics generation script.
- `predict.py`: Standalone inference script for single files.
- `app.py`: Streamlit web application.
- `notebook.ipynb`: Jupyter notebook containing the full workflow (equivalent to train + evaluate).
- `README.md`: Project documentation.
- `model/`: Directory containing the saved model (`best_model.keras`).
- `report.md`: Generated performance report (after running evaluation).
